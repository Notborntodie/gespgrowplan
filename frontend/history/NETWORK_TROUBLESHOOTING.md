# 网络连接问题排查指南

## 问题描述
当访问学习计划页面时，出现以下错误：
- `ERR_CONNECTION_REFUSED`
- `Network Error`
- 无法连接到服务器 `106.14.143.27:3000`

## 可能的原因和解决方案

### 1. 服务器未启动
**问题**：后端API服务器没有运行
**解决方案**：
```bash
# 检查服务器是否在运行
curl http://106.14.143.27:3000/api/health

# 如果服务器未启动，需要启动后端服务
# 请联系管理员启动服务器
```

### 2. 网络连接问题
**问题**：本地网络无法访问远程服务器
**解决方案**：
- 检查网络连接
- 尝试访问其他网站确认网络正常
- 检查防火墙设置

### 3. 服务器地址或端口错误
**问题**：API服务器地址或端口配置错误
**解决方案**：
- 确认服务器地址：`106.14.143.27`
- 确认端口：`3000`
- 联系管理员确认服务器配置

### 4. 临时解决方案
如果服务器暂时不可用，可以：
1. 使用模拟数据进行开发测试
2. 联系管理员获取服务器状态
3. 等待服务器恢复

## 测试连接
在浏览器控制台中运行以下代码测试连接：
```javascript
fetch('http://106.14.143.27:3000/api/health')
  .then(response => {
    if (response.ok) {
      console.log('✅ 服务器连接正常');
    } else {
      console.log('❌ 服务器响应异常:', response.status);
    }
  })
  .catch(error => {
    console.log('❌ 连接失败:', error.message);
  });
```
